Normal routing continues while CDRs are copied for ingest.

The diagram shows the OCN Node routing charging records between parties, while the EDX Connector sends a controlled copy to the CDR Ingest Service.

What the plugin does

Listens for CDRs

The plugin runs inside the OCN Node and watches for charging records that pass through the normal CDR endpoints.

Sends a copy

When a CDR is found, the plugin sends a copy to the CDR Ingest Service. The original routing continues normally.

Uses configured settings

The service URL, API key, enabled state and timeout are set during deployment. The API key is not printed in full.

Logs failures

If the ingest service is unavailable or rejects a CDR, the node logs the error and keeps running.
